### 第二步：打开你的“画布”——初始化Node.js项目

现在，我们打开你的电脑，开始写代码。我假设你已经安装了 `Node.js`。如果没有，请先去官网下载安装最新版。

**1. 创建项目文件夹**
在你的工作目录里，新建一个空文件夹，比如就叫 `grok-test`。然后打开终端（CMD、PowerShell 或者 VSCode 的终端），并把路径切换到该文件夹内。

**2. 初始化项目**
在终端里，输入以下命令并回车：

bash

这个命令会在文件夹里生成一个 `package.json` 文件。它就是项目的“身份证”。

**3. 安装所需依赖**
接着，安装一个非常好用的HTTP请求库——`axios`。在终端里输入：

bash

等待几秒钟，等待安装完成即可。

### 第三步：开始写核心代码！

现在，我们来写真正的调用代码。在项目文件夹里，新建一个文件，命名为 `grok.js`。

把下面的代码完整地复制进去。

javascript
// grok.js
const axios = require('axios');

// 你的千聚API Key（请务必替换成你自己的！）
const API_KEY = '这里替换成你上一步获取到的sk-开头字符串';

// 千聚的Base URL（固定写法，不要修改）
const BASE_URL = 'https://www.qianjuai.com/v1';

// 调用Grok4的函数
async function callGrok4(userMessage) {
  try {
    // 构建请求数据
    const requestData = {
      model: 'grok-4-latest', // 指定模型为Grok4最新版
      messages: [
        {
          role: 'user',
          content: userMessage, // 用户输入的消息
        },
      ],
      stream: false, // 设置为true可以开启流式输出，简单起见我们先关掉
    };

    console.log('正在向千聚API发送请求...');

    // 使用axios发送POST请求
    const response = await axios.post(`${BASE_URL}/chat/completions`, requestData, {
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${API_KEY}`,
      },
    });

    // 解析返回的数据
    const responseData = response.data;

    // 打印出AI的回复
    if (responseData.choices && responseData.choices.length > 0) {
      const aiMessage = responseData.choices[0].message.content;
      console.log('🤖 Grok4说：');
      console.log(aiMessage);
    } else {
      console.log('⚠️ 没有获取到回复内容，请检查请求。');
      console.log('返回的完整数据：', JSON.stringify(responseData, null, 2));
    }
  } catch (error) {
    console.error('❌ 请求发生错误：');
    // 打印更详细的错误信息
    if (error.response) {
      // 服务器返回了错误状态码
      console.error('状态码：', error.response.status);
      console.error('错误详情：', JSON.stringify(error.response.data, null, 2));
    } else if (error.request) {
      // 请求已发出，但没有收到响应
      console.error('网络无响应，请检查网络连接或Base URL是否正确。');
      console.error('错误信息：', error.message);
    } else {
      console.error('请求配置错误：', error.message);
    }
  }
}

// 测试运行：让Grok4做一个小测试
const testMessage = '你好！请用一行Python代码实现斐波那契数列。';

callGrok4(testMessage);

### 常见报错急救指南（Part 5）

如果你是零基础的“踩坑”大王，遇到下面的问题别慌，按图索骥：

**❌ 问题1：`fetch is not defined` 或 `Module not found`**
**原因**：你的 Node.js 版本太低，或者忘了装依赖。
**解决**：升级 Node.js 到 18以上版本，或者确保你执行过 `npm install axios`。

**❌ 问题2：`Error: 401 Unauthorized`**
**原因**：你的 API Key 写错了，或者没有正确放入代码中。
**解决**：重新检查 `grok.js` 文件中最开头那行 `const API_KEY = '...';`，确保引号内的字符串和你创建的Key完全一致。

**❌ 问题3：`Error: 400 Bad Request` 并且返回信息包含 `model not found`**
**原因**：你指定的模型名称不存在。虽然Grok4很新，但官方模型名称可能有细微变化。
**解决**：
1.  登录千聚官网（www.qianjuai.com）。
2.  进入控制台，找到“模型列表”或“文档”页面。
3.  搜索你刚刚指定的 `grok-4-latest`，看看它是否真的支持。如果不支持，换成一个正确的模型名，比如 `grok-3` 或其他可用模型。
4.  **另外**，请确保你的API Key余额不是负数。虽然送了$0.2，但可能之前被其他测试用光了。可以去控制台“账单”或“额度”页面查看。

**❌ 问题4：请求一直转圈圈，没反应，最后超时**
**原因**：最常见的原因是网络问题。
**解决**：
1.  你先检查自己是不是挂着代理或VPN？**强烈建议完全关闭所有代理软件**再试。
2.  如果还是不行，检查你代码里的 `BASE_URL` 是不是写成了 `https://api.grok.com`？**必须**改为我们提供的千聚地址：`https://www.qianjuai.com/v1`。这个地址在国内是直连的，非常快。
