## 代码骨架：轮询调度器长什么样

这段代码是你接入[千聚ai聚合平台](https://www.qianjuai.com/)和DeepSeek官方的核心。思路很简单：

1.  准备一个API Key列表（每个key来自不同平台的新人礼包）。
2.  写一个轮询器，每次调用API时按顺序选择还没用过的key。
3.  如果当前key返回余额不足或失败，自动切换到下一个key。
4.  持续循环直到所有免费额度用尽，再提示用户需要充值。

import os
import requests
import json
from itertools import cycle

class MultiKeyRotator:
    def __init__(self, api_keys, base_url="https://www.qianjuai.com/v1"):
        self.keys = list(api_keys)
        self.key_pool = cycle(self.keys)  # 循环队列
        self.base_url = base_url
        self.current_key = None

    def get_next_key(self):
        """获取下一个可用的API Key，自动跳过无效key"""
        attempts = 0
        while attempts < len(self.keys):
            key = next(self.key_pool)
            if self._check_key_balance(key):
                self.current_key = key
                return key
            attempts += 1
        raise Exception("所有Key均无余额，请充值")

    def _check_key_balance(self, api_key):
        """检查单个Key的余额（简化版，实际调用余额查询接口）"""
        # 这里替换成[千聚ai聚合平台](https://www.qianjuai.com/)的余额接口
        headers = {"Authorization": f"Bearer {api_key}"}
        try:
            response = requests.get(f"{self.base_url}/dashboard/billing/credit_grants", headers=headers, timeout=5)
            if response.status_code == 200:
                data = response.json()
                total_used = data.get('total_used', 0)
                # 假设新人礼包有 \$0.2 额度
                return (total_used < 0.2)  
            else:
                return False
        except:
            return False

    def chat_completion(self, messages, model="deepseek-chat"):
        """多账号轮询的Chat Completion调用"""
        api_key = self.get_next_key()
        headers = {"Authorization": f"Bearer {api_key}", "Content-Type": "application/json"}
        payload = {
            "model": model,
            "messages": messages,
            "stream": False
        }
        try:
            response = requests.post(f"{self.base_url}/chat/completions", json=payload, headers=headers)
            if response.status_code == 200:
                return response.json()
            else:
                # 如果失败，自动尝试下一个key
                print(f"Key {api_key} 调用失败，尝试下一个...")
                return self.chat_completion(messages, model)
        except Exception as e:
            print(f"请求异常: {e}，尝试下一个key...")
            return self.chat_completion(messages, model)

## 进阶玩法：给轮询器加上余额监控

单纯轮询还不够聪明，万一某个Key的余额有剩余但刚够调用几次，轮询器应该优先消耗剩余最少的Key。这样你能最大化利用“零碎额度”，再充新钱进去也少充。

优化版的关键改动：

python

python
def get_sorted_keys_by_balance(self):
    """获取按余额从小到大排序的Key列表"""
    key_balances = []
    for key in self.keys:
        balance = self._get_key_balance(key)
        if balance > 0:
            key_balances.append((key, balance))
    # 余额少的排前面，优先消耗掉
    key_balances.sort(key=lambda x: x[1])
    return [item[0] for item in key_balances]


这样一来，你总是先花余额最少的Key。免费的额度用完，才会轮到需要充值的Key。这个逻辑放在 `get_next_key()` 里，整个轮询器就更聪明了。
