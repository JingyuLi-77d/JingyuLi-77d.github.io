## 第二步：搭建 Grok4FastAPI 代理的代码骨架

现在开始动手。我们先用 FastAPI 写一个简单的代理服务。新建一个 Python 文件，比如 grok_fastapi_proxy.py，然后复制下面的代码：


from fastapi import FastAPI, HTTPException
from openai import OpenAI
import uvicorn
from pydantic import BaseModel

app = FastAPI(title="Grok4FastAPI Proxy")

# 配置千聚的 API 地址和你的 API Key
client = OpenAI(
    base_url="https://www.qianjuai.com/v1",
    api_key="这里填你在千聚获取的 API Key"
)

class ChatRequest(BaseModel):
    message: str

@app.post("/chat")
async def chat_with_grok(request: ChatRequest):
    try:
        response = client.chat.completions.create(
            model="grok-1",  # 这里换成 Grok 在千聚支持的模型名称
            messages=[{"role": "user", "content": request.message}]
        )
        return {"reply": response.choices[0].message.content}
    except Exception as e:
        raise HTTPException(status_code=500, detail=str(e))

if __name__ == "__main__":
    uvicorn.run(app, host="0.0.0.0", port=8000)


看到没有？代码里只有两处关键配置：
1. 把 `base_url` 改成了千聚的中转地址 `https://www.qianjuai.com/v1`
2. 把 `api_key` 换成了你在千聚拿到的 API Key

如果你之前用过 OpenAI 的 API，那你应该一眼就能看出来，这套接口格式完全兼容 OpenAI 标准。你在 OpenAI 上写好的代码，只需要改这两行，就能无缝切换到 Grok 的模型上。
