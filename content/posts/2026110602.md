## 进阶：实现流式输出（Stream）

聊天机器人、AI助手之类场景，最常见的效果是“打字机输出”——模型生成一个字，客户端的界面上就显示一个字。这需要用到流式请求。

把上面的`wenxin-chat.js`升级一下，添加流式支持。代码如下：

javascript
require('dotenv').config();
const axios = require('axios');

const API_KEY = process.env.API_KEY;
const BASE_URL = 'https://www.qianjuai.com/v1';

async function callWenXinStream() {
  const requestBody = {
    model: 'ernie-4.0-8k-latest',
    messages: [
      { role: 'user', content: '给我讲一个100字左右的睡前故事。' }
    ],
    stream: true // 开启流式输出
  };

  try {
    const response = await axios.post(`${BASE_URL}/chat/completions`, requestBody, {
      headers: {
        'Content-Type': 'application/json',
        'Authorization': `Bearer ${API_KEY}`
      },
      responseType: 'stream' // 告诉axios我们期待一个流式响应
    });

    console.log('文心一言正在回复：\n');
    response.data.on('data', (chunk) => {
      // 数据是Buffer格式，先转成字符串
      const lines = chunk.toString().split('\n');
      for (const line of lines) {
        // 流式响应通常以data: 开头
        if (line.startsWith('data: ') && !line.startsWith('data: [DONE]')) {
          const jsonStr = line.slice(6); // 去掉'data: '前缀
          try {
            const parsed = JSON.parse(jsonStr);
            const content = parsed.choices[0]?.delta?.content || '';
            // 逐字打印内容，模拟打字效果
            process.stdout.write(content);
          } catch (e) {
            // 如果解析失败则忽略
          }
        }
      }
    });

    // 监听流结束
    response.data.on('end', () => {
      console.log('\n\n--- 回复结束 ---');
    });

  } catch (error) {
    console.error('流式请求失败:', error.message);
  }
}

callWenXinStream();


跑起来之后，你会看到文字像在打字一样逐字输出。这个体验对于Chat UI来说至关重要。

**新手常见的坑：** 如果发现流式输出没有任何内容，大概率是你电脑上的Node版本过低，或者你用的代理工具拦截了流式响应。建议直接升级Node到最新LTS版本，并且确认代码里`responseType: 'stream'`这个参数写对了。

## 多轮对话功能实现

如果我们想把对话历史保留下来，实现多轮对话，messages数组就要发挥关键作用。

javascript
require('dotenv').config();
const axios = require('axios');

const API_KEY = process.env.API_KEY;
const BASE_URL = 'https://www.qianjuai.com/v1';

async function multiTurnChat() {
  // 维护一个对话历史数组
  const conversationHistory = [
    { role: 'system', content: '你是聪明的助手，会用简短的中文回答。' }
  ];

  const userInputs = [
    '你好',
    '请告诉我Node.js和Python的主要区别',
    '能给出一个简单的代码示例吗'
  ];

  for (const input of userInputs) {
    // 将用户的输入加入历史
    conversationHistory.push({ role: 'user', content: input });

    const requestBody = {
      model: 'ernie-4.0-8k-latest',
      messages: conversationHistory,
      temperature: 0.5
    };

    try {
      const response = await axios.post(`${BASE_URL}/chat/completions`, requestBody, {
        headers: {
          'Content-Type': 'application/json',
          'Authorization': `Bearer ${API_KEY}`
        }
      });

      const reply = response.data.choices[0].message.content;
      console.log(`用户: ${input}`);
      console.log(`助手: ${reply}`);
      console.log('---');

      // 将助手的回复也加入历史，这样下一次对话就能“记住”之前的上下文
      conversationHistory.push({ role: 'assistant', content: reply });

    } catch (error) {
      console.error(`第${userInputs.indexOf(input) + 1}轮对话失败:`, error.message);
    }
  }
}

multiTurnChat();


运行结果会展示三轮连贯的对话，你会发现文心一言能记住前面聊了什么，因为每次请求都把完整的历史`messages`数组传过去了。

**新手常见的坑：** messages数组太大，会消耗大量Token，导致成本激增。建议对生产环境设置对话轮次上限，比如只保留最近5轮对话。
