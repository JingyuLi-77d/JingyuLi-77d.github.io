### 第三步：写代码！100%成功的Python调用示例

这是文章的核心。我们将编写一个函数，用来识别一张图片里的内容。假设你有一张包含英文表格的截图，我们让Qwen-VL模型读取出里面的文字。

1.  **准备图片**
    先准备一张图片。为了方便测试，你可以使用一张网络上的图片，或者本地的一张 `/path/to/your/table.png`。
    -   **网络图片法**：直接使用图片的URL链接。
    -   **本地图片法**：将图片编码为Base64格式。这里为了方便演示，我们采用网络图片法。

2.  **编写完整的Python脚本**
    在你的编辑器里新建一个文件，命名为 `qwen_vl_test.py`，然后粘贴以下代码：

    python
    # qwen_vl_test.py
    import os
    from openai import OpenAI

    # 配置客户端 —— 核心操作！
    client = OpenAI(
        # 将API接口指向我们的API中转站，而不是OpenAI官方
        base_url="https://www.qianjuai.com/v1",
        # 读取环境变量中的密钥，避免明文泄露
        api_key=os.environ.get("QIANJU_AI_API_KEY"),
    )

    # 定义图片路径（这里用一张官方的示例表格图）
    image_url = "https://upload.wikimedia.org/wikipedia/commons/1/15/Simple_table.png"

    # 发起请求
    try:
        response = client.chat.completions.create(
            # 指定模型为 Qwen-VL-Plus 或 Qwen-VL-Max
            # Qwen-VL-Plus 速度快，性价比高；Qwen-VL-Max 精度更高
            model="qwen-vl-plus",
            messages=[
                {
                    "role": "user",
                    "content": [
                        {
                            "type": "image_url",
                            "image_url": {"url": image_url},
                        },
                        {"type": "text", "text": "请详细描述这张图片中的内容，并把所有文字信息提取出来。"},
                    ],
                }
            ],
            max_tokens=2000, # 设置最大输出长度，避免输出被截断
        )

        # 打印返回的结果
        print(response.choices[0].message.content)

    except Exception as e:
        print(f"调用失败，错误信息：{e}")
        print("请检查API Key是否正确，或账户余额是否充足。")
    

3.  **运行脚本**
    回到你的命令行，确保你之前已经设置了环境变量（或创建了`.env`文件并配置好），然后运行：

    bash
    python qwen_vl_test.py

### 常见报错与全程避坑

即使代码一样，新手也常会遇到一些问题。这里提前帮大家排查。

1.  **错误：`AuthenticationError: 401`**
    -   **原因**：API Key错误、未设置、或者没有验证邮箱。
    -   **解决方案**：
        1.  确认`base_url`写对了（是 `https://www.qianjuai.com/v1`，注意末尾的 `v1`）。
        2.  去[千聚api聚合站](https://www.qianjuai.com/)后台检查API Key是否已被禁用，或重新创建一个。
        3.  检查邮箱，确认账号已被激活。

2.  **错误：`RateLimitError: 429`**
    -   **原因**：请求过于频繁，或账户余额不足。
    -   **解决方案**：
        1.  查看账户余额。[千聚api聚合站](https://www.qianjuai.com/)虽然并发无限制，但需要账户里有足够的Token额度。如果是测试，$0.2赠送额度足够。
        2. 在代码中稍微增加一点 `time.sleep(1)` 的延时。

3.  **错误：`BadRequestError: 400` 或图片无法识别**
    -   **原因**：图片格式不受支持，或者图片URL无效（比如需要登录才能访问的私有链接），或者图片过大（超过20MB）。
    -   **解决方案**：
        1.  建议使用JPEG、PNG、WEBP、GIF格式的图片。
        2.  如果是本地图片，务必转换为Base64编码。
        3.  尽量在代码中将 `max_tokens` 设置得大一些，比如 `2000`，避免模型输出到一半卡住。

### 企业级接入优化建议

当你跑通上面这个基础例子后，就可以考虑正式部署了。这几个小技巧能让你的程序更强壮。

3.  **结果缓存**：对于同一张图片的反复查询（比如商品图片），建立一个本地缓存库，避免重复调用API产生费用。
