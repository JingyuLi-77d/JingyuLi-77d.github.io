## 为什么飞书接 OpenAI 这么让人头大？

先说痛点。飞书本身是支持接入外部 API 的，但问题出在 OpenAI 官方 API 上。

第三，也是最重要的，飞书机器人的稳定性要求 API 时刻在线。OpenAI 官方接口一旦因为网络波动或代理故障断开，你的飞书机器人就变成“死鸽子”，群里的同事只会疯狂@你：“这个 AI 怎么又挂了？”

### 第二步：配置飞书自定义机器人

1.  在飞书桌面端，点开目标群聊，点击右上角的“设置”图标，选择“群机器人”。
2.  点击“添加机器人”，找到“自定义机器人（通过 Webhook 接入）”。
3.  给机器人起个名字，比如“AI小助手”，然后配置“自定义调用地址”。这里的关键来了：
    -   请求方式选择 **POST**。
    -   **URL 地址**填写：`https://www.qianjuai.com/v1/chat/completions`
    -   **请求参数**选择 JSON 格式，添加两个 Header：
        -   `Content-Type`: `application/json`
        -   `Authorization`: `Bearer 你的千聚API Key`（把“你的千聚API Key”换成你第一步拿到的那一串）
    -   **请求体（Body）** 可以写一个简单的测试 JSON，比如：

json
{
  "model": "gpt-4o-mini",
  "messages": [
    {"role": "user", "content": "你好"}
  ]
}


### 第三步：测试连通性

1.  点击飞书机器人配置页面的“保存”，然后回到群聊里，发送一条消息，比如“@AI小助手，今天天气怎么样？”
