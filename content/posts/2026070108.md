## 全参数配置详解：保姆级拆解

很多教程只教怎么调通，不告诉你每个参数什么意思。这里我把调用 o3 时最常用的参数全拆开讲：

| 参数名 | 类型 | 说明 | o3 场景建议 |
|---|---|---|---|
| `model` | string | 模型名称，如 `o3-mini`，`o3-mini-2025-01-31` | 建议用 `o3-mini` 最稳定 |
| `messages` | array | 对话消息数组，支持 system / user / assistant 角色 | system 提示可复杂，o3 理解力强 |
| `temperature` | float | 采样温度，0-2 | o3 推荐用 0.2-0.5，扣严谨性 |
| `max_tokens` | int | 最大生成 Token 数 | o3 推理需要更多 token，建议设大（比如 4096） |
| `top_p` | float | 核采样概率，默认1 | 配合 temperature，一般不改 |
| `frequency_penalty` | float | 重复惩罚，-2到2 | 代码场景可适当开 0.1-0.3 |
| `presence_penalty` | float | 话题新鲜度惩罚 | 长对话可试试 0.2 |
| `stream` | bool | 是否流式输出 | 建议 `True`，首 token 更快 |
| `stop` | string/array | 停止词 | 可选，如 `["\n"]` 控制输出格式 |

**注意：** o3 系列的 `reasoning_effort` 参数（推理力度）在千聚的兼容接口中同样可用。你可以像这样指定推理强度：

python
response = client.chat.completions.create(
    model="o3-mini",
    reasoning_effort="high",   # 可选 low / medium / high
    messages=[...]
)


`high` 模式推理更彻底，但 token 消耗更大；`medium` 是性价比平衡点。
